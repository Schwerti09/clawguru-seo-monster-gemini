## Pass 1 results summary

- **Duration:** ~7 hours wall-clock across 30 locales.
- **Provider:** `aya-expanse:8b` local Ollama (`localhost:11434`).
- **Throughput:** ~100–150 seconds per 40-key batch on operator's GPU.
- **Failures:** zero data loss; one Ollama 500 auto-retried successfully on `es`.
- **Quality:** ~80 % production-ready (baseline confirmed on `sv` probe). Technical/ops strings clean; marketing/FAQ strings needed Pass 2 polish.

## Pass 2 results summary (2026-04-24)

- **Duration:** ~75 minutes wall-clock across 30 locales.
- **Provider:** `gemini-2.5-flash-lite` via Gemini API (1 key, 4.5 s inter-batch pacing).
- **Scope:** 462 hot-path keys per locale (hero, pricing, nav, common, footer, faq, home, homepage, academy) × 30 locales = 13,860 polished keys.
- **Failures:** 0 (first attempt hit 429 rate limits — resolved by switching from `gemini-2.0-flash` to `gemini-2.5-flash-lite` + 4.5 s inter-batch pause).

- **Quality:** native-speaker-level polish on all user-facing sections. Awkward aya-Pass-1 phrasings corrected, idioms fixed, tone calibrated to modern B2B SaaS.

## The translation pipeline

### CLI
```bash
npm run i18n:translate-aya                    # all 28 gap locales
npm run i18n:translate-aya sv fi th           # specific locales
DRY_RUN=1 npm run i18n:translate-aya sv       # translate but don't write
```

### Env vars
```
OLLAMA_URL             http://127.0.0.1:11434 (default)
OLLAMA_MODEL           aya-expanse:8b → :32b → qwen2.5:14b → first available
BATCH_SIZE             40 keys per call (default)
DRY_RUN=1              inspect without writing
```

### Crash-safety
Incremental writes — an abort mid-run loses only the current batch, not prior batches. Re-run continues where it left off.

## Hardcoded-i18n blocker (Phase 2 of i18n roadmap)

Even with 100 % dictionary coverage, ~283 component files hardcode strings:
```tsx
{locale === "de" ? "Text DE" : "Text EN"}
```
These **ignore** the dictionary. Full plan in [`i18n-roadmap-2026-04-22.md`](./i18n-roadmap-2026-04-22.md) Phase 2.

**Policy for new code (enforced Academy ∞ build):**
- Never inline-ternary on `locale === "de"`.
- Content modules colocated with the feature (see `lib/academy/hubContent.ts`).
- Or: `getDictionary(locale)`.
