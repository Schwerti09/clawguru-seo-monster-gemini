### Notes

- `de`'s 67 % is misleading in isolation — it's the secondary source of truth and many keys in the dictionary were authored in English first and never rephrased for German. In practice DE renders via inline ternaries in component code (see Phase 2 blocker in `i18n-roadmap-2026-04-22.md`).
- `roast.*` section (EN/DE-only by design) is excluded from the total.
- Short strings identical across languages (`"ClawGuru"`, URLs, emoji buttons) inflate the "not filled" count slightly — real usable coverage is 2–5 % higher than the percent column.
- Locales marked 🔴 weak are the same ones the aya pipeline will fill first when it fires.

## The translation pipeline

### What it does
`scripts/translate-via-aya.js`:
1. Reads `en.json` as reference, flattens to dot-keys.
2. For each target locale, compares to the locale's current dictionary.
3. Identifies missing keys (empty OR identical to EN OR absent).
4. Skips `roast.*`.
5. Batches 40 keys per Ollama call, strict JSON-mode responses.
6. Writes back incrementally — crash during a batch does not lose earlier batches.

### Provider chain
```
OLLAMA_MODEL env → aya-expanse:8b → aya-expanse:32b → qwen2.5:14b → first available
```

### CLI
```bash
# All 28 gap locales (excludes en, de by convention)
npm run i18n:translate-aya

# Specific locales
npm run i18n:translate-aya sv fi th

# Dry run — translate but don't write files
DRY_RUN=1 npm run i18n:translate-aya sv
```

### Cost / throughput
- Local Ollama: **zero API cost**, throughput limited by GPU/CPU.
- With aya-expanse:8b on a modern GPU: ~5–10 s per 40-key batch.
- Full fill of one locale (~200 missing keys): ~1–2 minutes.
- All 28 locales: ~30–60 minutes.

## Hardcoded-i18n blocker (Phase 2)

Even with 100 % dictionary coverage, ~283 component files hardcode strings as:
```tsx
{locale === "de" ? "Text DE" : "Text EN"}
```
These **ignore** the dictionary. Full plan in [`i18n-roadmap-2026-04-22.md`](./i18n-roadmap-2026-04-22.md) Phase 2.

**Policy for new code (enforced by the Academy ∞ build):**
- Never inline-ternary on `locale === "de"`.
- Content modules colocated with the feature (see `lib/academy/hubContent.ts` as the pattern).
- Or: pull strings from `/dictionaries/{lang}.json` via `getDictionary(locale)`.
