# Data Breach Response

## Objective
Confirm or refute unauthorized data access, minimize ongoing exposure, meet legal/regulatory obligations.

## First steps
- Engage legal/compliance.
- Preserve logs and evidence.
- Identify affected systems and datasets.

## Containment
- Disable compromised accounts/tokens.
- Block suspicious IPs/regions (as appropriate).
- Patch exploited vulnerabilities.

## Scope assessment
- Determine what data was accessed/exfiltrated.
- Use logs: DB audit, object storage access logs, API gateway logs.

## Notification
- Follow legal guidance.
- Prepare factual comms.

## Post-incident
- Improve monitoring for sensitive data access.
- Add anomaly detection.
- Reduce data retention.
